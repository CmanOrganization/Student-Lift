# Student Lift Architecture

## System Components

Frontend (Vercel)
    ↓
Backend API (Render)
    ↓
Database (MongoDB)

## URLs

Frontend: https://student-lift.vercel.app
Backend: https://student-lift.onrender.com
Database: MongoDB Atlas

## How It Works

1. User opens website in browser
2. Frontend loads from Vercel
3. User clicks "Request Ride"
4. Frontend sends request to Render Backend
5. Backend stores data in MongoDB
6. Backend sends response back to frontend
7. User sees confirmation

## Technologies Used

- Frontend: HTML, CSS, JavaScript
- Backend: Node.js, Express
- Database: MongoDB
- Hosting: Vercel (frontend), Render (backend)
- Version Control: GitHub
